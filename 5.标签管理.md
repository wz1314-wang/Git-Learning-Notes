发布一个版本时，我们通常先在版本库中打一个标签（tag），这样，就唯一确定了打标签时刻的版本。将来无论什么时候，取某个标签的版本，就是把那个打标签的时刻的历史版本取出来。所以，标签也是版本库的一个快照。
**标签tag**就是一个让人容易记住的有意义的名字，**它跟某个commit绑在一起**。
- 标签总是和某个commit挂钩。如果这个commit既出现在master分支，又出现在dev分支，那么在这两个分支上都可以看到这个标签。
- 创建的标签都只存储在本地，不会自动推送到远程。所以，打错的标签可以在本地安全删除。
# 创建标签
```bash
# 默认给最近一次commit打标签
git tag v1.0

# 给指定commitID打标签
git log --oneline
git tag v0.9 f52c633

# 创建带有说明信息的标签
git tag -a v0.8 -m "version 0.8 is released" 1094adb

# 查看所打的标签信息
git show v0.9

# 查看所有标签名称（注意，标签不是按时间顺序列出，而是按字母排序的）
git tag

# 筛选语法快速查找特定标签
git tag -l "v0.*"  # 列出所有 v0 开头的标签（如 v0.8、v0.9）
git tag -l "v1.*"  # 列出所有 v1 开头的标签（如 v1.0、v1.1）

```

# 操作标签
创建的标签都只存储在本地，不会自动推送到远程。所以，打错的标签可以在本地安全删除。
```bash
# 删除本地的某个标签
git tag -d v1.0

# 删除远程仓库origin的某个标签
git push origin --delete v0.9

# 推送某个本地标签到远程origin
git push origin v1.0

# 推送本地所有「尚未推送到远程 origin」的标签到远程origin
git push origin --tags

# 拉取远程所有标签到本地（本地没有的远程标签会被同步，本地已有的标签不变，也不会被覆盖）
git pull origin --tags

```
